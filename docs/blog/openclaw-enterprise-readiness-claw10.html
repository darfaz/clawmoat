ClawMoat raises the composite score from 1.2 to 2.2 ‚Äî not enterprise-ready yet, but a significant improvement. Let's look at the four dimensions where ClawMoat makes the biggest difference.

1. Authorization & Access Control (1 ‚Üí 3)

OpenClaw's current score: 1/5. There is no authorization layer. The agent runs with the user's full permissions.

With ClawMoat's Host Guardian, you get four permission tiers:

  • Observer ‚Äî read-only. Can list files and view system info. Cannot write or execute.
  • Worker ‚Äî safe commands (git, npm, file I/O). Blocks dangerous operations.
  • Standard ‚Äî most operations permitted. Forbidden zones enforced.
  • Full ‚Äî unrestricted execution. Forbidden zones still active. Full audit trail.

Plus 20+ forbidden zones that block access to sensitive directories regardless of tier: ~/.ssh, ~/.aws, ~/.gnupg, browser data, crypto wallets, package tokens, and more.

2. Audit Logging & Observability (2 ‚Üí 4)

OpenClaw's current score: 2/5. Basic session logging exists, but there's no structured audit trail for security events.

ClawMoat provides:

  • Full audit trail of every file access, shell command, and network request
  • Credential file monitoring (watches sensitive directories for unauthorized access)
  • Network egress logging with URL extraction and domain tracking
  • Real-time alerts via console, file, webhook, Slack, or email
  • Exportable logs for compliance review and incident forensics
  • Rate-limited alert delivery to prevent alert fatigue

This is the dimension where ClawMoat makes the biggest impact ‚Äî taking OpenClaw from "basic logs" to "structured, exportable, real-time observability."

3. Privilege Model (1 ‚Üí 3.5)

OpenClaw's current score: 1/5. No privilege model exists. The agent inherits the user's full privileges.

ClawMoat's permission tiers implement least privilege at the host level:

  • Tiered command blocking ‚Äî dangerous commands (rm -rf, chmod 777, etc.) blocked by tier
  • File system restrictions ‚Äî read/write access controlled per tier
  • Forbidden zones ‚Äî always blocked, regardless of tier
  • Runtime tier switching ‚Äî promote or demote without restart

4. Supply Chain Security (1 ‚Üí 2.5)

OpenClaw's current score: 1/5. Skills are installed from ClawHub with no signature verification.

ClawMoat's skill integrity checker provides:

  • Hash-based verification of installed skills
  • 14 suspicious pattern detectors (eval, exec, fetch to unknown domains, etc.)
  • CLI: clawmoat skill-audit ~/.openclaw/skills/
  • Detects tampering after installation

This doesn't solve the root problem (no signature verification in ClawHub), but it provides a detection layer that didn't exist before.

What ClawMoat Doesn't Fix

Honesty matters. ClawMoat doesn't address:

  • Identity & Authentication (1/5) ‚Äî OpenClaw has no agent identity system. This needs to be fixed upstream.
  • Data Isolation (1/5) ‚Äî The agent still runs in the user's environment. True data isolation requires VM-level separation.
  • Execution Sandboxing (1/5) ‚Äî ClawMoat restricts what the agent can do, but doesn't sandbox execution at the process level. (See ClawShell for process-level isolation.)
  • Compliance Certifications (1/5) ‚Äî ClawMoat generates compliance-ready reports, but isn't itself certified.
  • Vendor Support (1/5) ‚Äî We're open source. Enterprise support is available through our business plans.
